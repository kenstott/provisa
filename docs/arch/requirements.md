## Semantic Layer / Semantic Model


## GraphQL Alias vs Cypher Alias Separation

Two distinct namespaces for relationship naming:

- **`alias`** (Cypher rel type): stored in `relationships.alias`, uppercase snake-case (e.g. `OPENED_BY`). Used exclusively in Cypher MATCH clauses. Carried through compilation via `JoinMeta.cypher_alias`.
- **`graphql_alias`** (GraphQL field name): derived from target table name + cardinality at fetch time via `derive_graphql_alias()` in `db_queries.py`. Stored in `relationships.graphql_alias` (nullable; computed on read when NULL). Used exclusively as the GraphQL schema field name (e.g. `openedBy`, `customers`).

These must never be conflated. `alias` must not appear as a GraphQL field name fallback.

`derive_graphql_alias` rules: singularize target table name (inflect + round-trip validation to catch over-singularization), then pluralize for one-to-many or keep singular for many-to-one. Result is camelCase.

## Compiler & Schema

- **REQ-418** (2026-04-23): The SQL generator must be type-aware for JOIN predicates: it must omit `CAST` expressions when both sides of a JOIN condition are already the same type. Column type information must be propagated from `api_endpoints.columns`, table column metadata, and Trino cache column definitions into the SQL generator. Emitting `CAST(col AS VARCHAR) = CAST(col AS VARCHAR)` on same-type JOIN columns is a no-op in Trino (hash join, no indexes) but breaks index seeks on index-backed engines.
  Implementation: build a thin schema lookup `(source_id, table_name, column_name) → provisa_type` from `state.tables` once at startup and pass it into the SQL generator. At JOIN predicate time, query the lookup to compare types on both sides and emit `CAST` only when types actually differ. The lookup must handle aliasing (operationId vs alias) and the `_nf_` column naming convention using the same normalization already in `_lookup_ep`. The compiler already has `compiled.sources` and column refs so the lookup can be queried at the point of JOIN predicate generation in `sql_gen.py`.
  **Constraint:** A view created to import cross-domain data must represent a named business concept that domain stewards would recognise independently of any specific report. Views created solely to work around cross-domain constraints—with no genuine domain meaning—degrade the semantic model. Test: if the view would not exist without the report, it should not exist at all.

`JoinMeta.cypher_alias` bridges the Cypher rel type through compilation without polluting the GraphQL layer. `label_map.py` uses `cypher_alias` if present, else uppercases the GraphQL field name.

- **REQ-422** (2026-04-25): All parameter values exposed on an API-backed table must return the same response shape (columns). - **REQ-466** (2026-06-06): Dataset identity resolution requires centralized service. - **REQ-467** (2026-06-07): Query-language dataset name resolution via centralized identity service (extends REQ-466). - **REQ-468** (2026-06-11): Transpilation is a compiler responsibility, never a transport responsibility.
